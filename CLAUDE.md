# Fantasy Mundial 2026

## Project Goal

Build a comprehensive dataset of all FIFA World Cup 2026 players for the official FIFA Fantasy game (play.fifa.com/fantasy). Per player: fantasy position, price, starting XI likelihood, and career stats (goals, assists, games) for club + national team over the last 4 seasons.

### What is done

- **All 1248 active players** fetched from the official FIFA Fantasy API with:
  - `display_name`, `first_name`, `last_name`, `known_name`
  - `country`, `country_abbr`, `group`, `squad_id`
  - `fantasy_position` (GK/DEF/MID/FWD), `fantasy_price` ($M)
  - `percent_selected`, `total_points`, `avg_points`, `form`, `one_to_watch`
  - `fifa_id` (useful for cross-referencing with other data sources)
- **Starting XI likelihood** derived from `percent_selected` ranking within each country+position
  - Slots: 1 GK, 4 DEF, 4 MID, 2 FWD = 11 starters per team
  - Categories: "Likely Starter", "Maybe Starter", "Unlikely Starter"
- **Data exported** to CSV and Parquet via polars
- **Player stats extracted** (goals, assists, games per season, split club vs national team)
  from **API-Football** — see the dedicated section below. 1244/1248 players matched (99.7%).

## FIFA Fantasy API

Discovered undocumented API at `https://play.fifa.com/json/fantasy/`:

| Endpoint | Status | Returns |
|----------|--------|---------|
| `players.json` | OK | All players: id, name, squadId, position, price, status, stats, percentSelected |
| `squads.json` | OK | 48 teams: id, name, group, abbr, isEliminated |
| `rounds.json` | OK | 8 rounds: id, status, startDate, endDate, stage |
| `teams.json` | 403 | Blocked |
| `fixtures.json` | 403 | Blocked |
| `bootstrap.json` | 403 | Blocked |

Player record fields:
```json
{
  "id": 1,
  "firstName": "Rayan",
  "lastName": "Ait-Nouri",
  "knownName": null,
  "squadId": 1,
  "position": "DEF",
  "price": 4.9,
  "status": "playing",
  "percentSelected": 1.2,
  "fifaId": "...",
  "stats": { "totalPoints": 0, "avgPoints": 0, "form": 0 },
  "oneToWatch": false
}
```

Player `status` values: `"playing"` (1248 in active WC squad) or `"transferred"` (233 cut from final squad).

## Player Stats Extraction (API-Football)

Player goals/assists/games per season — split by **club** and **national team** — are sourced
from **API-Football** (api-sports.io). Kept in **separate tables** from the FIFA fantasy data
(joinable on `player_id`), so they can be explored independently.

- **Host / auth**: direct `https://v3.football.api-sports.io`, header `x-apisports-key`
  (NOT RapidAPI). Key read from `api_football_key.txt` or env `API_FOOTBALL_KEY`.
- **Plan**: requires **Pro ($19/mo, 7,500 req/day)** — the free tier blocks season 2025/26 and
  caps at 100 req/day. Full run is ~5,000 requests (fits one day). Cancel after = effectively one-time.
- **Seasons**: 2022/23, 2023/24, 2024/25, 2025/26 (API season = start year: 2022→2022/23 … 2025→2025/26).
- **Matching (no `fifaId` exists — it is null for all players)**: resolve each country's
  national-team id (`/teams`), pull its squad (`/players/squads`) for stable API player ids, and
  align to the FIFA roster by surname+initial within the country. Result: **1244/1248 matched**.
  A few countries need explicit NT-id overrides (`COUNTRY_AF_ID` in `fetch_apifootball.py`:
  Côte d'Ivoire=1501, Bosnia & Herzegovina=1113).
- **Stats per (player, season, competition, team)**: `games`, `started`, `minutes`, `goals`,
  `assists`, `tackles`, `interceptions`, `duels`, `duels_won`, `saves`, `goals_conceded`,
  `rating`. (`saves`/`goals_conceded` are null for outfield players — read the CSV with
  `pl.read_csv(..., infer_schema_length=None)` so they type as Int64, not str.) National-team comps
  (World Cup, confederation qualifiers, Nations League, Friendlies, etc.) come through as their
  own rows. **No ball-recovery and no xG/xA** — API-Football does not provide them.
- **Re-run**: `python run_apifootball.py` (responses cached in `apifootball_cache/`, so re-runs
  are fast/resumable). Outputs the three `*_apifootball.csv` files.

### Team records (national teams)

`run_team_stats.py` builds W/D/L + goals-for/against for all 48 national teams from the
**fixtures** endpoint (`/fixtures?team={ntid}&season={year}`, ~192 requests, same cache).
Per finished match (team's perspective) it derives goals for/against and result; a penalty
shootout counts as a **draw** (W/D/L decided on goals incl. extra time). Seasons 2022–2025.
Outputs `wc2026_team_stats.csv` (per team/season/competition) + `wc2026_team_stats_by_season.csv`
(per team/season). Note: raw totals favour teams that play more matches — use win-rate /
per-game rates and/or exclude `Friendlies` for fair comparison. Notebook **Section 7** loads these.

### Sofascore — abandoned

An earlier attempt used Sofascore's JSON API (via `curl_cffi`), which *did* have `ballRecovery`
and broad coverage, but it repeatedly hit long-lived Cloudflare IP blocks (403 "challenge") and
could not finish reliably. **Decision: skipped in favour of API-Football**, and all its artifacts
(scripts, partial CSVs, cache) were removed in the repo cleanup. **FBref was also ruled out** —
full Cloudflare JS challenge defeats requests / soccerdata / curl_cffi alike.

## Analytics app

`analytics.py` is the single source of truth for analysis logic (pure polars functions);
both the notebook and `app.py` import it. **Score metrics** in `rank_players`:
`sum goals and assists`, `goals only`, `assists only`, `interceptions`,
`weighted average interceptions, assists, goals` (0.05/0.25/0.65), `saves`,
`real multiplication of goals and assists` (**default**; assists x3 all positions; goals x4 FWD /
x5 MID / x6 DEF / x7 GK). Each metric's plain-language formula lives in `METRIC_FORMULAS` (single source
of truth, shown in the app's "How the scores are computed" expander). Position-dependent metrics
score each player by their **own** position, so
`position` may be one (`'FWD'`) or several (`['FWD','MID']`). Filters: `years` range on
`season_start_year`, `comp_type` (club/national, score-only), `country`, `starting_likelihood`,
`min_price`/`max_price`. `score_mode` is `per_game` (default) or `total`. Output columns:
player, nation, position, club (all clubs played for), market_value, starting_likelihood,
club_/nt_ games/goals/assists, interceptions, saves (if GK included), score. `top=None` = all.

Run the app: `streamlit run app.py` (Streamlit 1.58 + `streamlit-aggrid`, installed; tables use
AG Grid for Excel-like per-column filtering/sorting). Three tabs — **Player rankings**
(dropdowns + country filter), **Per-country** (records table + drill-down), **Player lookup**
(identity + club/national/total aggregate + expandable raw stats). Validate headlessly with
`streamlit.testing.v1.AppTest`.

## Technical Notes

- **Polars** is the primary DataFrame library (also exports to pandas)
- Python 3.12 on Windows 11
- Use `PYTHONIOENCODING=utf-8` if running Python from bash (cp1252 encoding issues)
- API data is cached locally as JSON files; delete them to re-fetch fresh data
- **`fifaId` is null for every player** in the FIFA Fantasy data — there is no external id key,
  so all cross-source matching is by name + national team (see API-Football matching above)
